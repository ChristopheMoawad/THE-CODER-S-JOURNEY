CSS Structure

CSS stands for Cascading Style Sheets. It is a stylesheet language used to describe the presentation of a document written in HTML or XML. CSS is used to control the layout, colors, fonts, and other visual aspects of a web page.

CSS is typically written in a separate file with a .css extension, and is linked to the HTML document using the <link> element in the <head> section of the HTML document. For example:

<link rel="stylesheet" type="text/css" href="styles.css">

CSS consists of rules that define how HTML elements should be displayed. Each rule has a selector and a declaration block. It is written as so:

selector { 
            property1: value; 
            property2: value; 
            property3: value;
        }

The selector specifies which HTML elements the rule applies to -- by type of element, class, or ID -- and the declaration block contains one or more declarations that specify the styles to be applied to those elements.

On that note, let us define what classes and IDs are:

  • Class: A class is a reusable style that can be applied to multiple elements. It is defined using a period (.) followed by the class name. For example, .my-class would define a class called "my-class". You can then apply this class to any HTML element by adding the class attribute to the element and setting its value to the class name. For example, <div class="my-class"> would apply the "my-class" styles to that div element.

  • ID: An ID is a unique identifier that can be applied to a single element. It is defined using a hash symbol (#) followed by the ID name. For example, #my-id would define an ID called "my-id". You can then apply this ID to an HTML element by adding the id attribute to the element and setting its value to the ID name. For example, <div id="my-id"> would apply the "my-id" styles to that div element.

Note that CSS rules are applied in a cascading manner, meaning that each element inherits styles from parent elements, and can have their own specific styles. In case of any conflicts, the more specific rule will be applied, and if multiple rules have the same specificity, the last one defined will take precedence.

Additionally, CSS allows combinations of selectors to create more complex and flexible styling rules. The most common combinations are defined as such:

  • selector1, selector2: selects the union of all elements matching either selector

  • selector1 ~ selector2: selects all elements matching selector2 that are following siblings of an element matching selector1

  • selector1 + selector2: selects all elements matching selector2 that are immediately next siblings of an element matching selector1

  • selector1 > selector2: selects all elements matching selector2 that are direct children of an element matching selector1

Other Ways to Include CSS

We have said that CSS is normally written in a separate file with a .css extension. But, it can also be included directly in the HTML document using the <style> element in the <head> section, or also using the style attribute directly on HTML elements. For example:

<head>
        <style>
                h1 {
                color: blue;
                }
                /* CSS rules go here */
        </style>
</head>

or

<h1 style="color: blue;">My Heading</h1>

However, it is generally recommended to use external stylesheets for better organization and maintainability of your code.

Visual Improvements

Box Model and Layout

To understand visual improvements in CSS, we need to first understand what elements are represented as: we call it the box model.

Every element in CSS is represented as a rectangular box, which consists of four parts: the content, padding, border, and margin.

  • The content is the actual content of the element, such as text or images.
  • The padding is the space between the content and the border, which can be used to create space around the content.
  • The border is a line that surrounds the padding and content, which can be styled with different colors, widths, and styles.
  • The margin is the space outside the border, which can be used to create space between elements.

Display and Positioning

These boxes can behave differently depending on the CSS properties applied to them.

For example, the display property can be used to change how an element is displayed. The most common values for this property are:

  • block: The element is displayed as a block-level element, which means it takes up the full width of its container and starts on a new line. Examples of block-level elements include <div>, <h1>, and <p>.
  • inline: The element is displayed as an inline element, which means it only takes up as much width as necessary and does not start on a new line. Examples of inline elements include <span>, <a>, and <strong>.
  • inline-block: The element is displayed as an inline-block element, which means it behaves like an inline element but can also have block-level properties such as width and height.
  • none: The element is not displayed at all, and does not take up any space on the page.
  • flex: The element is displayed as a flex container, enabling a one-dimensional layout model. Child elements are arranged along a single axis — either a row or a column — and can be flexibly aligned, spaced, and sized. It is best suited for laying out items in a single direction, such as navigation bars or button groups.
  • grid: The element is displayed as a grid container, enabling a two-dimensional layout model. Unlike flexbox, child elements are arranged across both rows and columns simultaneously, making it ideal for more complex page layouts such as dashboards or image galleries.
Learn more about display property

Another important property is the position property, which can be used to change how an element is positioned on the page. The most common values for this property are:

  • static: The element is positioned according to the normal flow of the document. This is the default value for all elements.
  • relative: The element is positioned relative to its normal position, with a certain offset defined by the top, right, bottom, and left properties.
  • absolute: The element is positioned relative to its nearest positioned ancestor,also using the top, right, bottom, and left properties.
  • fixed: The element is positioned relative to the viewport, which means it stays in the same position even when the page is scrolled.

Predefined Semantics and Generic Elements

We have seen that some elements have predefined semantics, which means they have a specific meaning in the context of the HTML document. Still, in some contexts we would like to use elemnts that do not have predefined semantics. This is where the div and span elements come in handy:

  • <div>: The div element is a block-level element that is used to group other elements together. It has no predefined semantics, which means it does not have any specific meaning in the context of the HTML document. It is often used as a container for other elements, and can be styled with CSS to create different layouts and designs.
  • <span>: The span element is an inline element that is used to group text or other inline elements together. Like the div element, it has no predefined semantics. It is often used to apply styles to specific parts of text or to group inline elements together for styling purposes.

Fonts, colors, alignments, units and sizing

CSS provides a wide range of properties for controlling the fonts, colors, and sizes of elements on a web page. Some of the most commonly used properties include:

  • font-family: This property is used to specify the font of an element. It can take a list of font names as its value, and the browser will use the first available font in the list. For example, font-family: Arial, sans-serif; would specify that the element should use the Arial font if it is available, and if not, it should use a generic sans-serif font.
  • color: This property is used to specify the color of text or other elements. It can take a variety of color values, including named colors (e.g., red, blue), hexadecimal color codes (e.g., #ff0000 for red), and RGB color values (e.g., rgb(255, 0, 0) for red).
  • background-color: This property is used to specify the background color of an element. It can take the same color values as the color property.
  • font-size: This property is used to specify the size of text.
  • text-align: This property is used to specify the horizontal alignment of text. It can take the values left, right, center, and justify.
  • vertical-align: This property is used to specify the vertical alignment of inline elements. It can take the values top, middle, bottom, and baseline.

To help with alignment, the margin property can be set to auto, which centers the element horizontally within its parent container. This can also be applied individually using margin-left, margin-right, margin-top, and margin-bottom.

CSS can use various units (absolute and relative) to set the size of elements.

  • px: This is an absolute unit that represents a fixed number of pixels. For example, font-size: 16px; would set the font size to 16 pixels.
  • cm/mm/pt/pc: These are absolute units that represent centimeters, millimeters, points, and picas respectively. For example, font-size: 1cm; would set the font size to 1 centimeter.
  • em: This is a relative unit that represents the font size of the parent element. For example, font-size: 2em; would set the font size to twice the font size of the parent element.
  • rem: This is a relative unit that represents the font size of the root element (usually the <html> element). For example, font-size: 1.5rem; would set the font size to 1.5 times the font size of the root element.
  • %: This is a relative unit that represents a percentage of the parent element's size. For example, width: 50%; would set the width of an element to 50% of its parent element's width.
  • vw/vh: These are relative units that represent a percentage of the viewport's width or height. For example, width: 50vw; would set the width of an element to 50% of the viewport's width.

CSS also allows to set constraints on the size of elements, such as maximum and minimum widths and heights with max-width, min-width, max-height, and min-height. They are used in case an element's size is relative to its parent or the viewport for example.

CSS Animations

CSS presents pseudo-classes that allow you to define different styles for different states of an element, such as when it is hovered over, focused, or clicked. They are written as such: selector-name:hover, selector-name:focus, selector-name:active.

By default, the new appearance take effectimmediately. The transition property allows to generate smooth change between two appearances.

Another way to animate elements is by using CSS @keyframes, which allow you to define a sequence of styles that an element will transition through over time. For example:

<style>
    @keyframes example {
        0% {
            background-color: red;
        }
        50% {
            background-color: yellow;
        }
        100% {
            background-color: green;
        }
    }
</style>

This code defines a simple animation that transitions the background color of an element from red to yellow to green over time.

Responsiveness

A website can be seen in a variety of devices and screen sizes. In order to provide a good user experience, websites must be designed to adapt to different screen sizes and orientations. We say that they must be responsive.

A responsive website adjusts its layout, content, and functionality based on the characteristics of the device being used to access it, such as the screen size, resolution, and orientation.

To create a responsive website, developers use a combination of techniques, including:

  • Relative units: This technique involves using relative units such as percentages, ems, or rems to define the size and position of elements. This allows the layout to adjust based on the screen size and resolution.

  • Media queries: This technique involves using CSS media queries to apply different styles based on the characteristics of the device being used to access the website. For example, developers can use media queries to apply different styles for mobile devices versus desktop devices.

    Media queries are written as such:

    <style>
    @media (max-width: 600px) {
        /* Styles for mobile devices */
    }
    @media (max-width: 900px) {
        /* Styles for tablet devices */
    }
</style>

    This media query applies the styles inside the curly braces only when the screen width is 600 pixels or less, which is typically the case for mobile devices.
