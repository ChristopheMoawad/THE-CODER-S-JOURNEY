HTML Structure

The HTML structure is the same for all web pages. It is divided into a head and a body section.

The head section contains information about the web page, such as the title and meta tags. It is not displayed on the web page.

The body section contains the content of the web page, such as text, images, and links. Everything that is displayed on the web page is placed within the body section.

Here is an example of the basic structure of an HTML document:

<!DOCTYPE html>

    <html lang="en">

    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <title>CSC_1S004_EP</title>
    </head>

    <body>
           <h1>My first webpage</h1>

           <p>This is HTML code.</p>

    </body>

    </html>

In this example, the HTML document starts with the doctype declaration, which tells the browser that this is an HTML5 document.
The html element is the root element of the document, and it contains the head and body sections.
The head section contains meta tags and the title of the web page, while the body section contains a heading and a paragraph of text.

In order to execute this code, you need to save it as an HTML file and open it in a web browser.
This can be done in many ways. First, we can write the code in a code editor, such as Visual Studio Code.
Then, we can save the file with a .html extension and open it in a web browser, using the file:// protocol.
Another way to open the file is to use a local web server. In Visual Studio Code, we can use the Live Server extension to start a local web server.
Once the server is running (by clicking the "Go Live" button at the bottom-right of the VS Code window), we can open the web page in a browser by navigating to http://localhost:5500/filename.html, where filename is the name of your HTML file.
Every change you make to the HTML file will be automatically reflected in the browser, making it easier to test and debug your code. Otherwise, you would have to refresh the browser every time you make a change to see the results.

HTML Elements

HTML elements are the building blocks of HTML pages. They are represented by tags, which are enclosed in angle brackets.
An HTML element typically consists of a start tag, content, and an end tag. The start tag indicates the beginning of the element, while the end tag indicates the end of the element.
For example, the following code creates a paragraph element:

<p>This is a paragraph.</p>

In this example, the start tag is <p>, the content is "This is a paragraph.", and the end tag is </p>.
The <p> tag is used to define a paragraph in HTML.
There are many different HTML elements, each with its own purpose and functionality. Some of the main HTML elements include:

  • <!--comments-->: Comment element, used to add comments to the HTML code. Comments are not displayed on the web page and are only visible in the source code. example: 

    <!-- This is a comment -->
  • <h1> to <h6>: Heading elements, used to define headings of different levels. example: 

    <h1>Main Heading</h1>
  • <a>: Anchor element, used to create hyperlinks. example: 

    <a href="https://www.example.com">Visit Example</a>
  • <img>: Image element, used to embed images in a web page. example: 

    <img src="image.jpg">
  • <div>: Division element, used to group other elements together. example: 

    <div>Content</div>
  • <span>: Span element, used to group inline elements together. example: 

    <span>Inline content</span>
  • <ul> and <ol>: Unordered and ordered list elements, used to create lists. example: 

    <ul>
    <li>Element of unordered list</li>
    <li>Second element</li>
        <ol> <!-- You can nest lists -->
            <li>First ordered element</li>
            <li>Second ordered element</li>
        </ol>
    <li>Third element</li>
</ul>
  • <table>: Table element, used to create tables. example: 

    <table>
    <tr>
        <td>Cell 1</td>
        <td>Cell 2</td>
    </tr>
</table>

There are many other HTML elements that can be used to create rich and interactive web pages. The exhaustive list of HTML elements can be found on Mozilla Development Network or also on W3Schools.

HTML Files and Links

Elements such as <a href="../page2.html"> and <img src="../images/image.jpg"> require a path/address to another resource. This address is typically a URL, that can point to another HTML page, to an image, to a data file (e.g. CSV, JSON, XLSX).

There are two main types of paths: absolute and relative. An absolute path is a complete URL that points to a specific resource on the web. For example, <a href="https://www.example.com"> is an absolute path that points to the homepage of the example website.

A relative path, on the other hand, is a path that is relative to the current document. It does not include the full URL, but instead uses a shorthand notation to indicate the location of the resource. For example, <a href="page2.html"> is a relative path that points to a file named "page2.html" in the same directory as the current document.

Relative paths can also use special characters to indicate the location of the resource. For example, <a href="../page2.html"> points to a file named "page2.html" that is located one level up from the current directory. <a href="./page2.html"> points to a file named "page2.html" that is located in the same directory as the current document.

Using relative paths can make it easier to manage links and resources within a website, especially when the website is moved to a different domain or directory. However, it is important to ensure that the relative paths are correct and that the resources they point to are available, otherwise the links will not work.

HTML Errors

Errors are generally grouped into different categories based on their nature and impact on the webpage's functionality.

Some common categories of HTML errors include:

  • Syntax errors: These errors occur when the HTML code does not follow the correct syntax rules. Examples of syntax errors include:
    • <div> without a closing tag: This error occurs when a <div> element is not properly closed with a corresponding </div> tag. This can lead to layout issues and unexpected behavior on the webpage.
    • <img> without a source attribute: This error occurs when an <img> element is missing the required "src" attribute, which specifies the URL of the image to be displayed. This can result in broken images or missing content on the webpage.
    • <a> without an href attribute: This error occurs when an <a> element is missing the required "href" attribute, which specifies the URL of the link destination. This can lead to non-functional links and a poor user experience.
    • <head> without a title element: This error occurs when the <head> section of an HTML document does not contain a <title> element, which provides a title for the webpage. This can affect search engine optimization (SEO) and make it difficult for users to identify the webpage in search results or browser tabs.
  • Runtime errors: These errors occur when the HTML code is syntactically correct but encounters issues during execution. Examples of runtime errors include:
    • Broken links: This error occurs when an <a> element points to a URL that is no longer valid or accessible. This can lead to 404 errors and a poor user experience.
    • Missing resources: This error occurs when an HTML element references a resource (such as an image, stylesheet, or script) that cannot be found or loaded. This can result in broken images, missing styles, or non-functional scripts on the webpage.
    • Cross-origin issues: This error occurs when an HTML element tries to access a resource from a different domain without proper permissions. This can lead to security errors and prevent the resource from being loaded or accessed.
  • Logic errors: These errors occur when the HTML code is syntactically correct but does not produce the intended result. Examples of logic errors include:
    • Incorrect nesting of elements: This error occurs when HTML elements are not properly nested within each other. For example, if a <div> element is opened inside a <ul> element but is not closed before the closing </ul> tag, it can lead to unexpected layout issues and rendering problems on the webpage.
    • Incorrect use of attributes: This error occurs when HTML attributes are used incorrectly or inappropriately. For example, using the "href" attribute on an <img> element instead of an <a> element can lead to non-functional links and a poor user experience.
    • Incorrect use of semantic elements: This error occurs when semantic HTML elements are used incorrectly or inappropriately. For example, using a <div> element instead of a <header> element for the header section of a webpage can lead to accessibility issues and make it difficult for screen readers to understand the structure of the webpage.

It is important to validate HTML code using tools such as the W3C Validator Service to identify and fix errors, ensuring that the webpage functions correctly and provides a good user experience.
