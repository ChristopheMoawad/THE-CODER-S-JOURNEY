JavaScript Structure

JavaScript is a high-level, interpreted programming language that is commonly used for web development. It is a versatile language that can be used to create interactive and dynamic web pages, as well as server-side applications and mobile apps.

JavaScript code is normally written in a text editor and saved with a .js file extension. To execute JavaScript code, it needs to be included in an HTML page using the <script> tag in the <head> section of the HTML document. For example:

<!DOCTYPE html>

<html lang="en">

<head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="stylesheet" type="text/css" href="style.css">
    <script src="script.js" defer></script>
    <title> My Webpage </title>
</head>

<body>
    <!-- Your HTML content goes here -->
</body>

</html>

This code snippet can be used as a template for all HTML pages.

To verify its functionality, you can run the JavaScript code in your browser's console.

To open the console, you can use the following keyboard shortcuts:

  • Chrome/Firefox: Press F12 or Ctrl + Shift + J
  • Safari: Press Ctrl + Shift + C

Or simply right-click on the page and select "Inspect" to open the developer tools.

JavaScript Basics

Here are some important notions of JavaScript:

  • Variables: Variables are used to store data in JavaScript. They can be declared using the var, let, or const keywords. Prefer const by default, use let if you need to reassign, and avoid var which has a wider, less predictable scope. For example: 
    const city = 'Paris';       // cannot be reassigned
let score = 0;              // can be reassigned
score = 42;
console.log(city, score);  // Paris 42
  • Strings: Strings can be declared with single or double quotes. The most powerful syntax is template literals (backtick quotes), which let you embed variables and expressions directly inside a string using ${...}. For example: 
    const name = 'Alice';
const age = 25;

// Old way: concatenation
const msg1 = 'Hello, ' + name + '!';

// Preferred: template literal
const msg2 = `Hello, ${name}! You are ${age} years old.`;
console.log(msg2);  // Hello, Alice! You are 25 years old.
  • Conditionals: Use if, else if, and else to run code based on conditions. Variables declared with let or const inside a block { } only exist within that block — this is called block scope. For example: 
    const temperature = 18;

if (temperature > 25) {
    console.log('Hot');
} else if (temperature >= 15) {
    console.log('Mild');
} else {
    console.log('Cold');
}
  • Loops: The for loop repeats code a set number of times. To iterate over all elements of an array, the for...of loop is simpler and more readable. For example: 
    // Classic for loop
for (let i = 0; i < 3; i++) {
    console.log(`i = ${i}`);
}

// for...of loop over an array
const colors = ['red', 'green', 'blue'];
for (const color of colors) {
    console.log(color);
}
  • Arrays: Arrays are ordered lists that can hold any mix of types. They are 0-indexed and grow dynamically. Declaring an array as const prevents reassignment of the variable, but you can still modify its contents. For example: 
    const fruits = ['apple', 'banana', 'cherry'];
console.log(fruits[0]);       // apple
fruits.push('date');          // add to end
console.log(fruits.length);   // 4
  • Objects (Dictionaries): JavaScript objects are collections of key/value pairs called properties. You access them with dot notation or bracket notation. Properties can be added dynamically, and objects can be nested inside each other. For example: 
    const player = {
    name: 'Hero',
    level: 5,
    position: { x: 10, y: 3 },
};

console.log(player.name);            // Hero
console.log(player.position.x);     // 10
player.health = 100;// add a property dynamically
  • Functions: Functions are reusable blocks of code. They can be defined with the function keyword or as arrow functions using =>. Arrow functions are preferred for short, inline logic. Functions are treated as regular values and can be stored in variables or objects. For example: 
    // Classic declaration
function greet(name) {
    return `Hello, ${name}!`;
}

// Arrow function
const square = (x) => x * x;

console.log(greet('Alice'));  // Hello, Alice!
console.log(square(5));       // 25

JavaScript Events

Classical programming runs functions one after the other in a predictable order. But when a program needs to react to a user — a click, a mouse movement, a key press — the timing and number of those interactions cannot be predicted in advance. Event programming is the approach used to handle this.

Event programming relies on two things: a listener, which waits for a specific event to occur on an element, and a callback function, which is the code that runs in response when the event happens.

Adding an Event Listener

The function addEventListener associates a listener and a callback to a DOM element. It takes two arguments: the event type as a string, and the callback function to call when that event occurs. For example:

// HTML: <h1>Click on me!</h1>

const title = document.querySelector('h1');
title.addEventListener('click', actionClicked);

function actionClicked(event) {
    console.log('clicked');
}

Here, title is the element being listened to. The listener waits for a 'click' event, and each time one occurs, actionClicked is called. The number and order of calls depend entirely on the user — they cannot be seen from the code itself.

Modifying the DOM on Events

A common response to an event is to modify the page itself — for instance, adding new elements to the DOM. Each click triggers the callback, so new elements accumulate with every interaction. For example:

// HTML: <h1>Click on me!</h1>  <div class="container"></div>

const title = document.querySelector('h1');
title.addEventListener('click', actionClicked);

function actionClicked(event) {
    const newElement = document.createElement('p');
    newElement.textContent = 'Clicked!';

    const container = document.querySelector('.container');
    container.appendChild(newElement);
}

Modifying CSS on Events

JavaScript can also modify the appearance of an element in response to an event by adding or removing CSS classes. The .classList property exposes two key methods: .add() to apply a class, and .remove() to take it away. This lets you define the visual states entirely in CSS, and simply toggle them from JavaScript. For example:

// CSS: .highlighted { color: yellow; font-weight: bold; }
// HTML: <h1>Click on me!</h1>

const title = document.querySelector('h1');
title.addEventListener('click', actionClicked);

function actionClicked(event) {
    title.classList.add('highlighted');    // applies the CSS class
    // title.classList.remove('highlighted'); // removes it
}

A useful companion is .toggle(), which adds the class if absent and removes it if present — handy for on/off states like showing and hiding a menu.

Remarks

  • You may encounter the syntax element.onclick = actionClicked, which is also valid. However, it only allows one handler per event type on an element. addEventListener is preferred as it supports multiple listeners on the same element.
  • The opposite of addEventListener is removeEventListener, which detaches an existing listener from an element.
  • JavaScript can handle many types of events, including: 'click', 'dblclick', 'mouseover', 'keydown', 'keyup', 'submit', 'resize', and more.

Widgets

A widget is a graphical element that can interact with the user — buttons, checkboxes, text entries, color pickers, and so on. HTML provides a set of basic widgets through the <input> tag, whose behavior can be finely controlled using JavaScript.

Widgets are selected from the DOM just like any other element, and events are attached to them with addEventListener. A button listens for 'click', while a text entry listens for 'change', which fires when the user finishes editing and leaves the field. The current value of any input widget is accessible via .value. For example:

// HTML: <input id="button" type="button" value="Click me">
//       <input id="textEntry" type="text">

const button = document.querySelector('#button');
button.addEventListener('click', buttonClicked);

function buttonClicked(event) {
    const textEntry = document.querySelector('#textEntry');
    console.log(`You typed: ${textEntry.value}`);
}

const textEntry = document.querySelector('#textEntry');
textEntry.addEventListener('change', textModified);

function textModified(event) {
    console.log(`Text modified: ${textEntry.value}`);
}

The widget <input type="color"> lets the user pick a color interactively. It fires an 'input' event — rather than 'change' — every time the value is updated, giving a live, dynamic response. The selected color is available as a hex string via .value, and can be applied directly to an element's inline style. For example:

// HTML: <span id="textInColor">Choose the color of this text.</span>
//       <input id="colorInput" type="color">

const textElement = document.querySelector('#textInColor');
const colorElement = document.querySelector('#colorInput');
colorElement.addEventListener('input', changeColorAction);

function changeColorAction(event) {
    textElement.style.color = colorElement.value;
}

When writing content into a DOM element from JavaScript, you have two options. .textContent treats the value as plain text — any HTML tags typed by the user are displayed as-is, never interpreted. .innerHTML on the other hand parses the string as actual HTML, allowing tags to be rendered. For example:

// HTML: <input id="textEntry" type="text">
//       <span id="textRender"></span>

const textEntry = document.querySelector('#textEntry');
textEntry.addEventListener('change', textModified);

function textModified(event) {
    const textRender = document.querySelector('#textRender');
    textRender.textContent = textEntry.value;  // safe: treated as plain text
    // textRender.innerHTML = textEntry.value; // interprets HTML tags
}

While .innerHTML is more powerful — it can generate rich HTML from a string — it is considered bad practice to pass user input directly into it. A user could inject arbitrary HTML or JavaScript, causing unexpected behavior. For instance, typing <img src=x onerror=alert("Hello")> into a field rendered with .innerHTML would trigger a popup. This is known as code injection. Though limited to the client side here, the same principle applied server-side can lead to serious security vulnerabilities. When in doubt, prefer .textContent.

JavaScript Animations

Animating something on a webpage means repeatedly calling a function that updates and redraws the result over time — even when the user is not interacting. Each individual call produces a single frame. To give the impression of smooth, continuous motion, the animation loop must run at least 25 frames per second (fps), meaning each frame must complete in under 40ms. Screens typically refresh at 60fps, so there is no benefit in drawing faster than that.

requestAnimationFrame

JavaScript provides the built-in function requestAnimationFrame to handle animation loops. It takes a user-defined function as argument and calls it automatically as soon as the system is ready to draw the next frame, synchronized to the screen's refresh rate. The function receives the time elapsed since the loop began (in ms) as its parameter. For example:

frame();

function frame(currentTime) {
    console.log(currentTime);   // ms elapsed since loop start
    requestAnimationFrame(frame);
}

Note that this is not a recursive call. The function frame runs normally to its end, and requestAnimationFrame simply tells the system to call it again when ready — handling synchronization automatically, unlike a plain loop.

Using currentTime, you can trigger time-dependent changes. Here a box moves across the screen and changes color and size after 2s and 4s:

// CSS: .box { position: fixed; width: 100px; height: 100px; background-color: red; }

const boxElement = document.querySelector('.box');
let leftPos = 0;

animationLoop();

function animationLoop(currentTime) {
    boxElement.style.left = `${leftPos}px`;
    leftPos = leftPos + 1;

    if (currentTime > 2000) boxElement.style.backgroundColor = 'green';
    if (currentTime > 4000) {
        boxElement.style.backgroundColor = 'yellow';
        boxElement.style.width = '200px';
        boxElement.textContent = 'Animated in JavaScript!';
    }

    requestAnimationFrame(animationLoop);
}

setInterval and clearInterval

Another way to call a function repeatedly is setInterval. Unlike requestAnimationFrame, it is not tied to the screen's refresh rate — instead, you specify a fixed delay in milliseconds between each call. It returns an ID that can later be passed to clearInterval to stop the loop entirely. For example:

let count = 0;

const intervalID = setInterval(tick, 1000);  // call tick() every 1000ms

function tick() {
    count += 1;
    console.log(`tick ${count}`);
    if (count === 5) {
        clearInterval(intervalID);  // stop after 5 ticks
    }
}

setInterval is suited for tasks that need to happen at a fixed, predictable rate regardless of rendering — polling a value, updating a clock, or triggering periodic logic. For visual animations however, requestAnimationFrame is always preferred, as it stays in sync with the screen and pauses automatically when the tab is hidden.

Interaction and Animation

Events and animation loops can be combined. User interactions update some state, and the animation loop reads that state every frame. A key rule: there is only ever one animation loop for the whole page — it updates everything. For example, here clicks create falling discs, each stored in an array that the loop updates every frame:

// CSS: .circle { background-color: cyan; width: 100px; height: 100px;
//               position: fixed; border-radius: 50%; }

const radius = 50;
const discs = [];   // stores { element, y } for each disc

document.addEventListener('click', createDisc);
animationLoop();

function createDisc(event) {
    const el = document.createElement('div');
    el.classList.add('circle');
    document.body.appendChild(el);
    const y = event.clientY;
    el.style.left = `${event.clientX - radius}px`;
    discs.push({ element: el, y });
}

function animationLoop() {
    for (const disc of discs) {
        disc.y += 2;
        disc.element.style.top = `${disc.y - radius}px`;
    }
    requestAnimationFrame(animationLoop);
}

Canvas

For more complex graphics, JavaScript provides the <canvas> element — a bitmap drawing surface. Drawing is not done directly on the element, but through a rendering context obtained with getContext. The '2d' context gives access to primitives: rectangles, lines, arcs, images, and more. For example:

// HTML: <canvas></canvas>

const canvasElement = document.querySelector('canvas');
const ctx = canvasElement.getContext('2d');

ctx.canvas.width  = 600;
ctx.canvas.height = 600;

// Filled rectangle
ctx.fillStyle = 'lightgray';
ctx.fillRect(0, 0, 600, 600);

// Stroked path (lines)
ctx.strokeStyle = 'red';
ctx.beginPath();
ctx.moveTo(500, 150);
ctx.lineTo(200, 300);
ctx.stroke();

// Circle (arc)
ctx.fillStyle = 'salmon';
ctx.beginPath();
ctx.arc(400, 400, 50, 0, 2 * Math.PI);
ctx.fill();

Sprites

A sprite is a series of images depicting an animation — a walking character, an explosion, and so on. They are usually stored as a grid of frames packed into a single image file. Displaying a sprite means drawing only the relevant portion of that image using ctx.drawImage, which accepts source offset and dimensions alongside destination coordinates. Cycling through each portion frame by frame inside a requestAnimationFrame loop produces the animation. For example:

const ctx = document.querySelector('canvas').getContext('2d');
const img = new Image();
img.src = 'bird.png';
img.onload = displaySprite;

let idx = 0;
const w = 240, h = 314;  // size of one frame

function displaySprite() {
    ctx.clearRect(0, 0, ctx.canvas.width, ctx.canvas.height);

    idx = (idx + 1) % 22;              // cycle through 22 frames
    const srcX = (idx % 5) * w;        // column in the sprite sheet
    const srcY = Math.floor(idx / 5) * h; // row

    ctx.drawImage(img,
        srcX, srcY, w, h,   // source: which frame to cut
        0, 0, w, h           // destination: where to draw it
    );

    requestAnimationFrame(displaySprite);
}

Web APIs

So far, the content of your webpages has been fully defined locally in HTML and JavaScript. But browsers can naturally communicate with external resources over the network — and Web APIs (Application Programming Interfaces) are the standard way to do this from code, allowing you to request and use data from remote servers automatically.

Querying a URL

The simplest form of Web API interaction consists in embedding parameters directly into a URL. For instance, a search query on a website can often be expressed as:

https://some-service.com/search?q=butterfly&lang=en

The part before the ? is the endpoint — the address of the resource. What follows are the query parameters, formatted as key=value pairs separated by &. Many public APIs work this way, making them easy to test directly in your browser's address bar.

JSON

When an API returns data rather than an image or a page, it needs a structured text format that both the server and the client can read. Two formats are commonly used: XML and JSON.

  • XML (eXtensible Markup Language) is an older, tag-based format — verbose but strict and well-validated.
  • JSON (JavaScript Object Notation) is a more recent, curly-bracket format, lighter and more readable. It maps directly onto JavaScript objects, making it the standard choice for web APIs today.

A JSON response looks like a JavaScript object literal — keys and values, nestable, supporting strings, numbers, arrays, and booleans:

{
    "name": "Alice",
    "age": 25,
    "scores": [14, 17, 12],
    "address": { "city": "Paris", "zip": "75001" }
}

Fetching Data with fetch

The built-in fetch function contacts a server at a given URL and runs a callback once the response arrives. Crucially, this happens asynchronously — the rest of your code keeps running while the request is in flight, and the callback is only triggered when the server replies. You cannot assume a precise moment when the data will be available.

The response from fetch is a raw HTTP object, not yet usable as JavaScript data. Calling .json() on it parses the response body into an actual JavaScript structure. Both steps are chained using .then(), which passes the result of one function into the next. For example:

const url = 'https://some-service.com/api/data';

fetch(url)
    .then(convertJSON)
    .then(handleData);

function convertJSON(response) {
    return response.json();   // parses JSON text into a JS object
}

function handleData(data) {
    console.log(data);         // data is now a regular JS object
}

console.log('fetch is called');  // this prints before handleData runs

Using the Data in the DOM

Once the data is a JavaScript object, it can be used like any other — looped over, read, and used to build DOM elements dynamically. This is how external data gets rendered as actual page content. For example, if the API returns a list of items under a "results" key:

const body = document.querySelector('body');

function handleData(data) {
    for (const item of data['results']) {
        const el = document.createElement('p');
        el.textContent = item['name'];
        body.appendChild(el);
    }
}
